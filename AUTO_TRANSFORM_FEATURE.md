# 宝可梦自动变身和排序功能

## 功能概述

本次更新增加了宝可梦自动变身和排序功能，让战斗更加流畅和策略性更强。

## 主要功能

### 1. 自动变身系统

- **触发时机**：每次进入战斗时自动触发
- **变身规则**：自动使用排在第一位的宝可梦牌进行变身
- **效果**：
  - 玩家立即变身为该宝可梦
  - 该宝可梦牌从抽牌堆中移除（本回合不再抽到）
  - 宝可梦的属性和技能立即生效

### 2. 宝可梦排序功能

- **访问方式**：在地图界面点击"查看宝可梦"按钮
- **排序方法**：拖拽宝可梦卡牌进行排序
- **视觉提示**：
  - 第一张卡牌有绿色边框和发光效果
  - 第一张卡牌显示"⭐ 首发出战"标记
  - 其他卡牌显示序号（#2, #3...）

### 3. 战斗策略

- **首发选择**：将最强的宝可梦放在第一位
- **战术搭配**：根据敌人属性选择克制的宝可梦首发
- **后续战斗**：在战斗中仍可使用其他宝可梦牌进行切换

## 技术实现

### 修改的文件

1. **js/pokemon-card-system.js**
   - 添加 `reorderPokemonCards()` 方法：处理卡牌排序
   - 添加 `getFirstPokemonCard()` 方法：获取第一张宝可梦牌
   - 添加 `autoTransformOnBattleStart()` 方法：战斗开始自动变身
   - 添加 `handleDragStart/Over/Drop/End()` 方法：处理拖拽排序
   - 更新 `renderPokemonView()` 方法：添加拖拽和视觉提示

2. **js/battle.js**
   - 修改 `startBattle()` 函数：添加自动变身调用

3. **js/game.js**
   - 在 GameState.player 中添加 `pokemonCards` 数组
   - 在 `resetGame()` 中初始化宝可梦牌组

4. **js/ui.js**
   - 修改 `selectStarterPokemon()` 函数：初始化宝可梦牌组

5. **index.html**
   - 更新宝可梦查看界面：添加排序提示

## 使用示例

### 战斗开始流程

```
1. 进入战斗
   ↓
2. 系统自动检查 pokemonCards 数组
   ↓
3. 获取第一张宝可梦牌
   ↓
4. 自动使用该牌变身
   ↓
5. 从抽牌堆移除该牌
   ↓
6. 战斗开始
```

### 排序操作流程

```
1. 点击"查看宝可梦"按钮
   ↓
2. 拖拽卡牌到目标位置
   ↓
3. 释放鼠标
   ↓
4. 系统更新排序
   ↓
5. 界面重新渲染
```

## 注意事项

1. **首次游戏**：选择初始宝可梦后，该宝可梦会自动成为第一位
2. **捕捉新宝可梦**：新捕捉的宝可梦会添加到列表末尾
3. **空列表处理**：如果没有宝可梦牌，战斗仍正常开始，只是没有自动变身
4. **多宝可梦切换**：自动变身后，仍可在战斗中使用其他宝可梦牌
